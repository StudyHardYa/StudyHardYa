HashMap源码：

结论：

先说下结论，有的小伙伴可能没有心思看看完！

数据结构：

加载因子为0.75f

默认初始容量为16

扩容为原来的1倍 <<1 临界点也会随着增加1倍 <<1

依靠每次在数组中添加成功都会自增的size与临界点做比较，是否需要扩容

如果添加key相同的键值对会进行value的替换，并返回替换掉的value值

链表未大于等于8位时：

​ 数组加链表，其实就是一个Node类型的数组，而而每个下表都是Node类，Node它的结构为链表的形式，记录下一个、本身键值对，为单向链表

转换红黑树条件：

​ 数组长度大于64，并且链表长度大于8才会进行转换

​

与HashTable的区别：

JDK1.8 ：

HashMap在添加时进行数组实例化，HashTable实例化时就会进行数组实例化

HashTable不会转换为红黑树

线程安全性不同。HashMap线程不安全；Hashtable 中的方法是Synchronize的。
key、value是否允许null。HashMap的key和value都是可以是null，key只允许一个null；Hashtable的key和value都不可为null。
迭代器不同。HashMap的Iterator是fail-fast迭代器；Hashtable还使用了enumerator迭代器。
hash的计算方式不同。HashMap计算了hash值；Hashtable使用了key的hashCode方法。
默认初始大小和扩容方式不同。HashMap默认初始大小16，容量必须是2的整数次幂，扩容时将容量变为原来的2倍；Hashtable默认初始大小11，扩容时将容量变为原来的2倍加1。
是否有contains方法。HashMap没有contains方法；Hashtable包含contains方法，类似于containsValue。
父类不同。HashMap继承自AbstractMap；Hashtable继承自Dictionary。

HashMap初始化：

HashMap有着多个重载的构造方法，接下来 一 一来看看分别有着什么作用！！

调用的构造方法

 public HashMap(int initialCapacity, float loadFactor) {
     	//判断当前设置的默认容量是否为0 ，为0抛出异常
        if (initialCapacity < 0)
            throw new IllegalArgumentException("Illegal initial capacity: " +
                                               initialCapacity);
     	//判断是否超出最大容量，如果超出就赋予最大容量
        if (initialCapacity > MAXIMUM_CAPACITY)
            initialCapacity = MAXIMUM_CAPACITY;
     	//判断当前加载因子是否小于等于0 Float.isNaN这个方法目前还不是很清楚
        if (loadFactor <= 0 || Float.isNaN(loadFactor))
            throw new IllegalArgumentException("Illegal load factor: " +
                                               loadFactor);
        this.loadFactor = loadFactor;  //为加载因素赋值
        this.threshold = tableSizeFor(initialCapacity);  //根据当前容量得出下次需要扩容的临界点
}

static final int hash(Object key) {
        int h;
        return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
    }

   		// 1、 tab为当前map数组，p为当前节点，n为数组长度，i为当键值对在数值的下表位置
        Node<K,V>[] tab; Node<K,V> p; int n, i;

        if ((tab = table) == null || (n = tab.length) == 0)

            n = (tab = resize()).length;

        if ((p = tab[i = (n - 1) & hash]) == null)

            tab[i] = newNode(hash, key, value, null);
        else {
            Node<K,V> e; K k;
            if (p.hash == hash &&
                ((k = p.key) == key || (key != null && key.equals(k))))
                e = p;
            else if (p instanceof TreeNode)
                e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
            else {
                for (int binCount = 0; ; ++binCount) {
                    if ((e = p.next) == null) {
                        p.next = newNode(hash, key, value, null);
                        if (binCount >= TREEIFY_THRESHOLD - 1) // -1 for 1st
                            treeifyBin(tab, hash);
                        break;
                    }
                    if (e.hash == hash &&
                        ((k = e.key) == key || (key != null && key.equals(k))))
                        break;
                    p = e;
                }
            }
            if (e != null) { // existing mapping for key
                V oldValue = e.value;
                if (!onlyIfAbsent || oldValue == null)
                    e.value = value;
                afterNodeAccess(e);
                return oldValue;
            }
        }
    	//6、modCount进行计数  这个便利店作用显目有解释
        ++modCount;
    	//7、size++ 判断当前加入数组中的数量是否大于临界值
        if (++size > threshold)
            resize();
        afterNodeInsertion(evict);
        return null;
    }
HashMap初始化及调整大小方法：

​ 这个方法主要是为了当我们数组为null没有进行初始化时进行初始化，并设置临界值，

final Node<K,V>[] resize() {
    Node<K,V>[] oldTab = table; //3.1、hashMap数组
    int oldCap = (oldTab == null) ? 0 : oldTab.length;  //3.2、旧数组大小，如果数组为空则为0 ，不为空则为数组大小
    int oldThr = threshold; //3.3、临界点 旧
    int newCap, newThr = 0;  //3.4、newCap 数组大小 新    newThr：临界点 新
    if (oldCap > 0) { //3.5、判断旧数组大小是否大于0 第一次不会进入
        //判断旧容量是否为最大值了
        if (oldCap >= MAXIMUM_CAPACITY) {
            //如果数值为最大值了就将最大值为临界点
            threshold = Integer.MAX_VALUE;
            //并返回就数组回去，不进行扩容
            return oldTab;
        }
        //进行扩容参数初始化，新长度为原来的1倍，新临界点也为原来的1倍
        else if ((newCap = oldCap << 1) < MAXIMUM_CAPACITY &&
                 oldCap >= DEFAULT_INITIAL_CAPACITY)
            newThr = oldThr << 1; // double threshold
    }
    else if (oldThr > 0) //3.6、判断临界值 旧 是否大于0 第一次不会进入
        newCap = oldThr;
    else {
        //3.7、进行基础数据初始化
        newCap = DEFAULT_INITIAL_CAPACITY; //将默认容量大小 16 赋予新数组大小
        //新数组容量与加载因子进行计算得出 临界点 并赋值为临界点局部变量 新
        newThr = (int)(DEFAULT_LOAD_FACTOR * DEFAULT_INITIAL_CAPACITY);
    }
    //3.8判断临界值 新 是否为0  一般情况下不会进入
    if (newThr == 0) {
        float ft = (float)newCap * loadFactor;
        newThr = (newCap < MAXIMUM_CAPACITY && ft < (float)MAXIMUM_CAPACITY ?
                  (int)ft : Integer.MAX_VALUE);
    }
    //3.9、将局部临界点赋值给全局临界点
    threshold = newThr; 
    
    	//3.10、初始化新map数组 长度为局部newCap大小 
        Node<K,V>[] newTab = (Node<K,V>[])new Node[newCap];
     //3.11、将新map数组
    table = newTab; 
    //3.12、判断旧map数组是否为空 第一次不会进入
    if (oldTab != null) {
        for (int j = 0; j < oldCap; ++j) {
            Node<K,V> e;
            if ((e = oldTab[j]) != null) {
                oldTab[j] = null;
                if (e.next == null)
                    newTab[e.hash & (newCap - 1)] = e;
                else if (e instanceof TreeNode)
                    ((TreeNode<K,V>)e).split(this, newTab, j, oldCap);
                else { // preserve order
                    Node<K,V> loHead = null, loTail = null;
                    Node<K,V> hiHead = null, hiTail = null;
                    Node<K,V> next;
                    do {
                        next = e.next;
                        if ((e.hash & oldCap) == 0) {
                            if (loTail == null)
                                loHead = e;
                            else
                                loTail.next = e;
                            loTail = e;
                        }
                        else {
                            if (hiTail == null)
                                hiHead = e;
                            else
                                hiTail.next = e;
                            hiTail = e;
                        }
                    } while ((e = next) != null);
                    if (loTail != null) {
                        loTail.next = null;
                        newTab[j] = loHead;
                    }
                    if (hiTail != null) {
                        hiTail.next = null;
                        newTab[j + oldCap] = hiHead;
                    }
                }
            }
        }
    }
    //返回map数组 新
    return newTab;
}

modCount作用：

Fail-Fast 机制
我们知道 java.util.HashMap 不是线程安全的，因此如果在使用迭代器的过程中有其他线程修改了map，那么将抛出ConcurrentModificationException，这就是所谓fail-fast策略。这一策略在源码中的实现是通过 modCount 域，modCount 顾名思义就是修改次数，对HashMap 内容的修改都将增加这个值，那么在迭代器初始化过程中会将这个值赋给迭代器的 expectedModCount。在迭代过程中，判断 modCount 跟 expectedModCount 是否相等，如果不相等就表示已经有其他线程修改了 Map：注意到 modCount 声明为 volatile，保证线程之间修改的可见性。

添加至同一个下标中：

此处是第二次添加，第一次添加于上面无差别

注意，要两个key的Hash值相同

主方法：
transient Node<K,V>[] table;


//执行方法
final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
                   boolean evict) {
   		// 1、 tab为当前map数组，p为当前节点（会随着循环链表成为循环到的那个结构），n为数组长度，i为当键值对在数值的下表位置
        Node<K,V>[] tab; Node<K,V> p; int n, i;      
    	//2、此处判断的同时也为我们的tab指向了table  n值为tab的长度 由于不为空 不进入该方法
        if ((tab = table) == null || (n = tab.length) == 0)
            n = (tab = resize()).length;
    	// 3、获取当前数组长度减一 并使用 ‘与’ 逻辑运算符获取 数组下表位置赋值给i 并获取当前下表下的内容赋值给p，判断是否为空
    	//此次hash碰撞，下表相同，进行else
        if ((p = tab[i = (n - 1) & hash]) == null)
            tab[i] = newNode(hash, key, value, null);
        else {
            //4、e 局部e为临时存储变量，会一直改变 ，  k为当前数值中需要对比节点的key
            Node<K,V> e; K k; 
            if (p.hash == hash && //5、判断当前比较的节点的hash值与本次添加的hash值是否相同
                //此处为k进行赋值并判断他们的值是否相同 
                ((k = p.key) == key || (key != null && key.equals(k))))
                //如果key相同，将当前节点赋值给我们的e，下面会将我们要添加的与当前的value进行替换
                e = p;
            //6、判断当前是否为树节点
            else if (p instanceof TreeNode)
                e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
            else {
                //7、无限循环
                for (int binCount = 0; ; ++binCount) {
                    //8、将局部e赋值为当前数组上的节点的下一个节点，并判断是否为空
                    if ((e = p.next) == null) {
                        //9、创建一个节点赋值在当前循环到的节点的下一个节点
                        p.next = newNode(hash, key, value, null);
                        //10、判断当前循环长度是否大于等于8为，注意从0开始的，如果为true则转换为红黑树
                        if (binCount >= TREEIFY_THRESHOLD - 1) // -1 for 1st
                            treeifyBin(tab, hash);
                        break;
                    }
                    //11、判断循环中的节点是否与我们当前要添加的key相同，相同就终止循环
                    if (e.hash == hash &&
                        ((k = e.key) == key || (key != null && key.equals(k))))
                        break;
                    //不一样就将当前遍历到的节点赋值给p进行下一轮遍历
                    p = e;
                }
            }
            //判断e是否为null，不为null
            if (e != null) { // existing mapping for key
                //将原来的旧值赋值给我们的oldValue，等会将这个值返回给我们
                V oldValue = e.value;
                if (!onlyIfAbsent || oldValue == null)
                    //将我们新添加的value赋给当前节点的value进行替换
                    e.value = value;
                afterNodeAccess(e);
                //返回旧值
                return oldValue;
            }
        }
    	//5、modCount进行计数  这个便利店作用显目有解释
        ++modCount;
    	//6、size++ 判断当前加入数组中的数量是否大于临界值
        if (++size > threshold)
            resize();
        afterNodeInsertion(evict);
        return null;
    }

链表超出8位：

多余的就不多看了，都是上面一样了，我们直接看转红黑树的过程

如果数组小于64则不会进行红黑树的转换，还是进行扩容

final void treeifyBin(Node<K,V>[] tab, int hash) {
    int n, index; Node<K,V> e;
    //判断数组长度是否小于64，小于则是进行扩容
    if (tab == null || (n = tab.length) < MIN_TREEIFY_CAPACITY)
      //小于64于则进行扩容，还不会进行红黑树转换
        resize();
    else if ((e = tab[index = (n - 1) & hash]) != null) {
        TreeNode<K,V> hd = null, tl = null;
        do {
            TreeNode<K,V> p = replacementTreeNode(e, null);
            if (tl == null)
                hd = p;
            else {
                p.prev = tl;
                tl.next = p;
            }
            tl = p;
        } while ((e = e.next) != null);
        if ((tab[index] = hd) != null)
            hd.treeify(tab);
    }
}
